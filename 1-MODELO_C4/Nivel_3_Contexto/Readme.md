# ⚙️ Nivel 3: Diagrama de Componentes — TRANS-SYNC

Este directorio contiene el diseño detallado de **Nivel 3 (Componentes)** bajo el enfoque del Modelo C4 para el sistema **TRANS-SYNC**[cite: 1]. En este nivel se profundiza en la estructura interna de cada uno de los microservicios del ecosistema, detallando su organización interna en capas lógicas (Controladores, Servicios y Repositorios) y sus respectivas responsabilidades de negocio[cite: 1].

## 📂 Estructura del Directorio

Tal como se detalla en el árbol de archivos (ver `image_299379.png`), este módulo almacena los siguientes componentes de modelado arquitectónico[cite: 1]:

*   **📄 `c4_nivel3_pasajes.puml`:** Código fuente PlantUML que describe la estructura interna del microservicio de pasajes (gestión de rutas, mapa de asientos y emisión de boletos)[cite: 1].
*   **📄 `c4_nivel3_encomiendas.puml`:** Modelado que detalla los componentes del servicio de logística (registro de carga, trazabilidad de envíos y generación de QR)[cite: 1].
*   **📄 `c4_nivel3_auth.puml`:** Diagrama que desglosa el control perimetral de accesos, lógica de generación/validación de tokens JWT y seguridad de sesiones[cite: 1].
*   **📄 `c4_nivel3_usuarios.puml`:** Detalla la lógica de perfiles de empleados, asignación de choferes a rutas y gestión de roles/permisos[cite: 1].
*   **📄 `c4_nivel3_auditoria.puml`:** Estructura interna del microservicio de liquidación, encargado del cierre de caja, registro de logs transaccionales y análisis de anomalías financieras[cite: 1].
*   **📄 `Readme.md`:** Documentación guía del directorio actual[cite: 1].

---

## 🔍 Patrón de Diseño Estructural de los Componentes

Todos los microservicios aquí modelados siguen de manera estricta un patrón de arquitectura en capas y respetan los principios de diseño estructural aplicados en el proyecto[cite: 1]:

1.  **🎮 Capa de Controladores (Controllers):** Componentes expuestos (`@RestController`) encargados de recibir las peticiones HTTPS/JSON redirigidas por el API Gateway[cite: 1].
2.  **🧠 Capa de Servicios (Services):** Núcleo donde reside la lógica de negocio pura de la empresa de transportes, orquestando las reglas operativas y desacoplada del protocolo de red[cite: 1].
3.  **🗄️ Capa de Repositorios (Repositories/Data Access):** Interfaces de acceso de datos encargadas de la comunicación directa con la base de datos privada de cada microservicio a través de protocolos seguros[cite: 1].
